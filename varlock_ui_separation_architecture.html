Rules for staying decoupled

Do

Parse varlock's JSON output as your data model. If varlock adds a new field to its JSON, your UI gets it for free. Treat the JSON schema as the contract.

Don't

Parse varlock's human-readable terminal output with regex. That format can change in any release and will break silently.

Do

Use --cwd flag to point varlock at the correct project directory. This means one varlock installation serves all projects your UI manages.

Don't

Reimplement type validation, secret resolution, or env merging logic in your UI. If Varlock adds a new type or plugin, you'd have to update your code too.

Do

Version-check varlock on startup (varlock --version) and warn the user if it's below your minimum supported version.

Don't

Depend on varlock being installed globally. Bundle a specific varlock version with your app, or auto-install it if missing, so users don't hit "command not found."
