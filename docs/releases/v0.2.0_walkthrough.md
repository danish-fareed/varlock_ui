# Devpad Release v0.2.0 Walkthrough

This document outlines the changes, verification steps, and release notes for version `v0.2.0` of Devpad.

## Release Summary
We opted to transition directly to `v0.2.0` (skipping the alpha) in order to comply with the Windows MSI packaging restrictions, which require numeric-only, standardized version strings without alpha suffixes during bundling. 

This release introduces significant functionality, primarily centered around **deep terminal integration** and **UI consistency**.

## Changes Made
- **Native OS Terminal Attachment:** We implemented deep OS-level terminal integrations behind the scenes in Tauri. Users can now open native terminal windows (Windows Terminal, PowerShell, CMD, macOS Terminal, or Linux variants) directly at the project directory.
- **Run in Terminal:** The `CommandCard` now features an interactive "Run in Terminal" action, which spawns the command inside the user's native terminal emulator.
- **UI Layout Overhaul:** Refined the unified `Sidebar` layout, streamlined `AppLayout` and `DashboardPage` components, and resolved minor responsive issues. 
- **Fixed Testing Suite:** Addressed a failing unit test in [src/lib/utils.test.ts](file:///d:/github/varlock_ui/src/lib/utils.test.ts) where a generic `SECRET_MESSAGE` incorrectly matched against the strict `_SECRET` suffix logic.
- **Documentation:** Implemented a new [CHANGELOG.md](file:///d:/github/varlock_ui/CHANGELOG.md) file following the **Keep a Changelog** specification.

## Verification & Validation
### Automated Validation
- **Lint & Build:** `npm run tauri build` completed cleanly, bundling raw React code into optimized chunks alongside compiled rust `.exe` and `.msi` targets.
- **Test Suite:** The Vitest suite was successfully run (`npm run test -- --run`). After patching the [isSensitiveKey](file:///d:/github/varlock_ui/src/lib/utils.ts#1-20) utility, all tests passed.

### Manual Verification
- **Smoke test** performed on the built executable (`Devpad_0.2.0_x64-setup.exe`). The terminal attachment logic properly bridges between the React frontend UI and the Rust backend, ensuring stability in a real desktop environment.
